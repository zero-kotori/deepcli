# ADR 0001：Harness-first 重构

## 状态
已接受。

## 背景
deepcli 功能堆叠较全，但命令、schema、UI、runtime、产品报告和文档过度集中在少数大文件，复杂度入口集中、文档易漂移。

## 决策
下一阶段不再增加细碎命令，而是把项目收束为"核心链路小而精、行为可验证、文档可同步、边界清晰"。先提供轻量架构 harness（`docs/HARNESS.md`：模块地图、边界原则、文档同步与验证要求），再做去硬编码、代码拆分、文档瘦身、删除策略与 UI 收束，分阶段可验证、可回滚。

## 影响
- 模块边界与文档同步成为实现契约的一部分；行为变更需同步对应文档。
- 重构按 `docs/ai/HARNESS_REFACTOR_PLAN.md` 的阶段推进。
