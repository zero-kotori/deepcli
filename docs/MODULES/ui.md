# UI 模块

## 职责

`src/ui.rs` 负责终端 UI：消息输入、对话记录渲染、输入区临时浮层、resume 选择器、审批交互、运行中任务停止/恢复 runtime，以及动态状态渲染。`src/ui/dialogs.rs` 拥有统一 dialog owner：`DialogKind`、`TuiDialog`、`DiffDialog`、`AgentEditorDialog`、`SettingsDialog`、`InterviewDialog`、`open_diff_dialog`、`open_agent_editor_dialog`、`open_latest_agent_editor_dialog`、`open_settings_dialog`、`open_interview_dialog`、`render_dialog`、`dialog_view_for_state`、`dialog_body_for_state`、`handle_dialog_key`、`replace_dialog_field` 和 `clamp_selected_blocker_to_monitor`；权限弹窗、diff dialog、agent editor、settings 和采访弹窗共享这套渲染、焦点、保存和关闭路径。`src/ui/monitor.rs` 拥有 monitor tab catalog、tab metadata、静态 quick-action projection、运行态 quick-action projection、quick-action 行追加、monitor-only 动态 quick actions（`tool_quick_actions`、`deliver_quick_actions`、`environment_quick_actions`、`environment_action_target`、`environment_needs_setup`），以及只消费 `SessionMonitor` 的 tab formatter（`format_usage_tab_lines`、`format_deliver_tab_lines`、`format_tests_tab_lines`、`format_session_tab_lines`、`format_context_tab_lines`、`format_environment_tab_lines`、`format_approvals_tab_lines`）。`src/ui/monitor_shell.rs` 拥有 task monitor shell owner：`render_task_monitor`、`format_task_monitor_text`、`format_task_overview_lines`、`monitor_quick_actions_for_tab`、`monitor_tab_strip`、`format_monitor_tabs`、`select_monitor_tab_at_position`、`clicked_monitor_quick_action_index`、`visible_panel_line_indices`、`truncate_panel_lines`、`truncate_panel_lines_with_focus` 和 `selected_monitor_quick_action_line`；`src/ui.rs` 只保留外层键鼠分发、运行状态字段和 worker/runtime 编排。`src/ui/worker.rs` 拥有 worker drain owner：`WorkerDone`、`drain_progress` 和 `drain_done`；`src/ui.rs` 仍负责启动 worker 和发送 input。`src/ui/runtime_lifecycle.rs` 拥有 runtime lifecycle owner：`stop_running_task`、`mark_active_session_paused` 和 `rebuild_runtime_for_active_session`。`src/ui/monitor_changes.rs` 拥有 Changes tab 的 workspace/session diff projection、worktree snapshot 与 patch 交互 owner（`WorkspaceChangesSnapshot`、`WorkspaceDiffSection`、`handle_changes_tab_key`、`select_change_patch_at_row`、`refresh_workspace_changes_snapshot`、`format_changes_tab_lines`、`append_workspace_changes_lines`、`load_workspace_changes_snapshot`、`parse_git_status_snapshot`、`parse_diff_sections`）。`src/ui/monitor_tools.rs` 拥有 Tools tab 的工具日志、工具详情 projection 和选择/展开/快捷键交互 owner（`ToolLogItem`、`ToolTabLine`、`handle_tools_tab_key`、`prefill_tools_session_command`、`toggle_selected_tool`、`toggle_tool_at_row`、`move_selected_tool_by`、`select_tool_at_index`、`visible_tool_index_at_line`、`selected_tool_panel_line`、`format_tool_tab_lines`、`tool_tab_lines`、`append_tool_quick_action_lines`、`tool_detail_preview_lines`、`tool_detail_is_truncated`）。`src/ui/monitor_health.rs` 拥有 Health tab 的 workspace/config projection（`format_health_tab_lines`、`health_quick_actions_for_state`、`provider_needs_credentials_for_ui`、`provider_env_key_for_ui`、`presence_label`）。`src/ui/monitor_library.rs` 拥有 Library tab 的 workspace/library projection（`format_library_tab_lines`、`library_quick_actions_for_state`、`format_library_item`）。`src/ui/monitor_output.rs` 拥有 Result/Trace advanced output projection（`format_result_tab_lines`、`result_output_window_size`、`format_trace_tab_lines`）。

`src/ui.rs` 现在是 UI entrypoint final orchestration boundary：它只保留 `TuiState`、`run_basic_repl`、`run_tui`、`run_tui_loop`、`handle_tui_mouse`、`handle_tools_scroll_mouse`、`handle_tui_key` 和 `cycle_monitor_tab`，负责 TUI 生命周期、终端模式切换、状态字段持有、主事件循环、键鼠/paste/clipboard 分发与 owner 委派；业务投影、输入状态机、quick action、paste、clipboard、geometry、scrolling、worker drain 和 monitor render 均由 `src/ui/*` owner 承担。

`src/ui/command_palette.rs` 拥有 slash command palette owner：`COMMAND_PALETTE_MATCH_LIMIT`、`RUNNING_SAFE_PALETTE_PRIORITY`、`handle_command_palette_key`、`handle_command_palette_mouse_for_state`、`clicked_command_palette_index`、`command_palette_selection_event`、`complete_selected_command`、`clamp_selected_command`、`slash_command_suggestions_for_state`、`prioritize_running_safe_suggestions`、`running_safe_palette_priority`、`slash_command_query`、`render_command_palette`、`format_command_palette_text`、`command_palette_match_token` 和 `command_palette_matches_line_index`。`src/ui.rs` 只保存当前选中命令索引，并在输入焦点、键鼠事件与工具区域渲染时委派给该 owner。

`src/ui/credential_prompt.rs` 拥有 credential prompt owner：`CredentialPrompt`、`CredentialPromptSpec`、`handle_tui_credential_set_for_state`、`handle_credential_prompt_key`、`confirm_credential_prompt`、`parse_tui_credential_set`、`credential_prompt_hidden_body` 和 `credential_prompt_hidden_cursor`。`src/ui.rs` 只保存当前 credential prompt 状态，并在 `/credentials set`、prompt 按键处理、隐藏输入渲染和光标定位时委派给该 owner。

`src/ui/chat_view.rs` 拥有 chat view owner：`ChatUiLayout`、`chat_ui_layout`、`transcript_visible_message_count`、`transcript_window`、`format_transcript_text`、`format_messages_title`、`render_chat_ui` 和 `message_box_cursor_position`。`src/ui.rs` 只负责外层 TUI 事件循环和 owner 委派，主聊天布局、消息 transcript、输入区临时浮层和输入框光标定位由该 owner 编排。

`src/ui/chat_history.rs` 拥有 chat history owner：`ChatLine`、`const TUI_HISTORY_MESSAGE_CHARS`、`chat_lines_from_runtime`、`session_messages_to_chat_lines` 和 `truncate_history_message`。`src/ui.rs` 只保存当前聊天行状态，并在启动、resume 和测试中调用该 owner 从 session/runtime 投影历史消息。

`src/ui/session_projection.rs` 拥有 active session / header / task monitor projection owner：`ActiveSessionRef`、`HeaderStatus`、`active_session_ref`、`sync_active_session_ref`、`session_monitor_for_state`、`header_status_for_state`、`load_active_session_header`、`load_active_session_monitor`、`session_monitor_from_session`、`summarize_plan_for_tui` 和 `workspace_for_state`。`src/ui.rs` 只保存当前 active session 引用，并在启动、worker 完成、runtime rebuild 和 monitor 渲染时消费该 owner。

`src/ui/native_terminal.rs` 拥有默认原生终端聊天渲染：session header、彩色 `user` 输入标签、raw-mode 方向键输入编辑、assistant delta 流式输出、provider turn 状态线，以及每轮 provider 调用内折叠后的工具进度摘要；该路径不进入 alternate screen，也不恢复旧的 `>` 提示符或 Messages/Message Box/Task Monitor 面板。

`src/ui/message_box.rs` 拥有 message box input owner：`MessageBoxAction`、`MessageBox`、`handle_prompt_input_key`、`handle_key` 和 `insert_str`。`src/ui.rs`、credential prompt、approval prompt 和 command palette 只消费该输入状态机，不直接维护 buffer/cursor/history 编辑规则。

`src/ui/dashboard.rs` 拥有 dashboard snapshot owner：`TuiSnapshot` 和 `render_dashboard`。`src/ui.rs` 只注册该 owner 并 re-export 公开 API，不直接维护非交互 dashboard 布局或渲染。

`src/ui/running_commands.rs` 拥有运行中本地命令 owner：`handle_running_tui_local_command`、`running_tui_supported_command_hint`、`running_tui_deferred_input_hint`、`ensure_running_no_output`、`ensure_running_completion_is_observation_only`、`ensure_running_round_is_read_only`、`ensure_running_benchmark_is_read_only`、`ensure_running_preflight_is_planned`、`ensure_running_session_is_read_only`、`ensure_running_git_is_read_only`、`handle_tui_running_git`、`format_tui_running_status`、`handle_tui_running_btw` 和 `handle_tui_running_terminal`。`src/ui.rs` 只负责把 TUI 状态交给该 owner；`/stop` 后的 worker abort、session pause 和 runtime rebuild 由 `src/ui/runtime_lifecycle.rs` 拥有。

`src/ui/worker.rs` 拥有 worker drain owner：`WorkerDone`、`drain_progress` 和 `drain_done`。`src/ui.rs` 只保存 worker handle，调用该 owner 消费 progress/done channel，并继续拥有 worker spawn 和 input submit。

`src/ui/runtime_lifecycle.rs` 拥有 runtime lifecycle owner：`stop_running_task`、`mark_active_session_paused` 和 `rebuild_runtime_for_active_session`。`src/ui.rs` 和 running command owner 只调用该 owner 执行 `/stop` / running `/quit` 的 worker abort、session paused 状态写回和交互 runtime rebuild。

`src/ui/input_submission.rs` 拥有 input submission owner：`submit_tui_input`、`handle_tui_local_command` 和 `apply_resume_result`。`src/ui.rs` 只在主输入提交和 quick action 提交时调用该 owner；resume picker owner 复用 `apply_resume_result` 应用恢复结果。

`src/ui/text.rs` 拥有 shared UI text helper owner：`format_optional_u64`、`format_optional_bytes`、`format_cache_hit_rate`、`format_latest_environment`、`compact_ui_text`、`format_action_event`、`latest_action_result_line`、`latest_action_result`、`non_empty_output_lines`、`first_non_empty_line` 和 `short_id`。`src/ui.rs` 只导入这些 helper 供现有 owner 复用，不直接维护通用文本截断、短 ID、最新 action 输出摘要或 usage/environment 格式化逻辑。

`src/ui/scrolling.rs` 拥有 transcript/result scrolling owner：`TRANSCRIPT_SCROLL_STEP`、`TRANSCRIPT_MOUSE_SCROLL_STEP`、`RESULT_SCROLL_STEP`、`RESULT_MOUSE_SCROLL_STEP`、`handle_transcript_scroll_key`、`handle_result_scroll_key`、`scroll_result_from_mouse`、`scroll_result`、`scroll_result_down`、`result_scroll_event`、`result_output_line_count`、`scroll_transcript` 和 `transcript_scroll_event`。`src/ui.rs` 只在键鼠事件分发中调用该 owner，不直接维护消息历史或 Result 输出的滚动状态机。

`src/ui/quick_actions.rs` 拥有 monitor quick action activation owner：`handle_monitor_quick_action_key`、`selected_quick_action_event`、`activate_selected_monitor_quick_action` 和 `activate_monitor_quick_action_at_row`。`src/ui.rs` 只在键盘和鼠标事件分发时调用该 owner，不直接维护 quick action 选择、edit-before-run、提交和点击激活规则。

`src/ui/paste.rs` 拥有 paste routing owner：`handle_tui_paste` 和 `normalize_pasted_text`。`src/ui.rs` 只在 crossterm paste event 中调用该 owner；credential prompt、BTW answer prompt、resume filter、主 message box 和 Tools detail 预填复用同一换行归一化规则。

`src/ui/terminal_mode.rs` 拥有主 TUI terminal mode owner：setup/teardown 命令列表、滚轮/点击捕获、bracketed paste 和 alternate screen 进入/退出都由该 owner 维护；主 TUI 默认接管鼠标滚轮，让滚轮滚动 deepcli 对话内容而不是终端历史 scrollback，同时不启用 all-motion 鼠标追踪，也不启用 kitty/CSI-u 等增强键盘协议。

`src/ui/clipboard.rs` 拥有 clipboard owner：`handle_clipboard_key` 和 `handle_input_selection_mouse`。message box / BTW answer 的鼠标选区、键盘选区复制、OSC 52 clipboard 请求和 base64 编码由该 owner 维护；可被终端上报的 `Cmd-C` 与 `Ctrl-C` 共享同一复制路径。

`src/ui/geometry.rs` 拥有 geometry helper owner：`rect_contains` 和 `rect_content_row_contains`。`src/ui.rs` 只在外层鼠标事件分发时导入使用，command palette、resume picker、approvals、Changes、monitor shell 和 quick action owner 复用同一 hit-test helper。

`src/ui/tests.rs` 拥有 UI 大型单测模块。`src/ui.rs` 只保留 `#[cfg(test)] mod tests;`，不再内嵌大型测试模块。

`src/ui/resume_picker.rs` 拥有 resume picker owner：`ResumePicker`、`ResumeSelection`、`pick_resume_session`、`session_matches_resume_query`、`run_resume_picker_loop`、`resume_filter_accepts_char`、`handle_resume_picker_key`、`handle_resume_picker_mouse_for_state`、`handle_resume_picker_mouse`、`resume_picker_layout`、`render_resume_picker` 和 `format_resume_preview_text`。`src/ui.rs` 只保存当前 picker 状态、调用 owner 的键鼠处理与渲染，并在恢复结果返回后复用入口层的 session/runtime 状态同步。

`src/ui/approvals.rs` 拥有 approval interaction owner：`SideQuestionPrompt`、`ApprovalPromptView`、`SelectedBlocker`、`approval_prompt_active`、`approval_prompt_view_for_state`、`handle_approval_prompt_key`、`handle_approval_tab_key`、`handle_approvals_mouse_for_state`、`clicked_approvals_tab_index`、`selected_blocker`、`activate_selected_blocker`、`deny_selected_blocker`、`open_side_question_answer_prompt`、`handle_side_question_prompt_key`、`confirm_side_question_prompt`、`answer_side_question_for_state`、`update_selected_approval`、`update_approval_for_state` 和 `clamp_selected_blocker_to_monitor`。`src/ui.rs` 只保存当前审批/旁路问题选择状态，并把键盘/鼠标事件委派给该 owner；BTW 回答的可见弹窗由 `src/ui/dialogs.rs` 的 Interview dialog 渲染和收集输入。

## 边界

- UI 应渲染领域状态并收集用户意图；它不应定义权威的命令、工具、权限或会话契约。
- `src/ui.rs` 是 UI entrypoint final orchestration boundary；除 `TuiState`、`run_basic_repl`、`run_tui`、`run_tui_loop`、`handle_tui_mouse`、`handle_tools_scroll_mouse`、`handle_tui_key` 和 `cycle_monitor_tab` 外，不应新增直接业务投影、状态机或 helper 函数。
- 权限弹窗、diff dialog、agent editor、settings 和采访弹窗的渲染、焦点切换、按键保存和输入区覆盖行为由 `src/ui/dialogs.rs` 拥有；`src/ui.rs` 不直接维护这些 dialog 的正文拼接或保存逻辑。
- 运行安全的命令分发必须与实际的处理器支持及命令文档保持一致；运行中可用命令提示从 `CommandRouter::help_summaries()` 的 `running_safe` metadata 派生，只在 UI 层补充 `read-only`、`--dry-run` 等交互限制标签；运行中 slash palette 的优先级由 `RUNNING_SAFE_PALETTE_PRIORITY` 显式 projection 定义，并用测试约束它覆盖所有 registry running-safe 命令。
- slash command palette 的查询、running-safe 排序、匹配数量上限、键鼠选择、点击命中、完成输入和渲染由 `src/ui/command_palette.rs` 拥有；`src/ui.rs` 不直接维护 palette 查询、文本拼接或命令完成逻辑。
- credential prompt 的 `/credentials set` 解析、隐藏输入框打开、按键处理、保存 API key、隐藏正文和隐藏光标计算由 `src/ui/credential_prompt.rs` 拥有；`src/ui.rs` 不直接解析 credentials set 参数或保存 provider API key。
- chat view 的整体布局、消息 transcript 窗口、主聊天渲染和 message-box 光标定位由 `src/ui/chat_view.rs` 拥有；`src/ui.rs` 不直接拼接主聊天视图或维护输入区域布局。
- chat history 的行模型、session/runtime 消息转换和长历史截断由 `src/ui/chat_history.rs` 拥有；`src/ui.rs` 不直接维护 `ChatLine`、`chat_lines_from_runtime`、`session_messages_to_chat_lines` 或 `truncate_history_message`。
- active session 引用、header 状态、SessionMonitor fallback、plan 摘要和 workspace fallback 由 `src/ui/session_projection.rs` 拥有；`src/ui.rs` 不直接维护 `ActiveSessionRef`、`HeaderStatus`、`session_monitor_for_state`、`header_status_for_state`、`session_monitor_from_session` 或 `workspace_for_state`。
- message box 的输入 buffer、cursor、history、提交动作、粘贴插入和 prompt 输入键处理由 `src/ui/message_box.rs` 拥有；`src/ui.rs` 不直接维护 `MessageBox` / `MessageBoxAction` 的编辑状态机。
- dashboard snapshot 的数据结构和非交互渲染由 `src/ui/dashboard.rs` 拥有；`src/ui.rs` 不直接维护 `TuiSnapshot` 或 `render_dashboard`。
- worker progress/done channel 的 drain、工具日志写入、运行结果写回和 `WorkerDone` envelope 由 `src/ui/worker.rs` 拥有；`src/ui.rs` 不直接维护 `drain_progress`、`drain_done` 或 `WorkerDone`。
- 运行中任务停止、session paused 状态写回和恢复交互 runtime 的 rebuild 由 `src/ui/runtime_lifecycle.rs` 拥有；`src/ui.rs` 不直接维护 `stop_running_task`、`mark_active_session_paused` 或 `rebuild_runtime_for_active_session`。
- 输入提交、非运行中本地 TUI 命令（`/resume`、`/rename`、`/credentials set` 委派）和 resume 结果应用由 `src/ui/input_submission.rs` 拥有；`src/ui.rs` 不直接维护 `submit_tui_input`、`handle_tui_local_command` 或 `apply_resume_result`。
- 通用 UI 文本 helper、usage/environment 格式化、action 输出摘要、短 ID 和非空输出行提取由 `src/ui/text.rs` 拥有；`src/ui.rs` 不直接维护 `format_optional_u64`、`format_optional_bytes`、`format_cache_hit_rate`、`format_latest_environment`、`compact_ui_text`、`format_action_event`、`latest_action_result_line`、`latest_action_result`、`non_empty_output_lines`、`first_non_empty_line` 或 `short_id`。
- transcript/result 滚动常量、键盘滚动处理、鼠标 Result 滚动处理、滚动事件文案和输出行计数由 `src/ui/scrolling.rs` 拥有；`src/ui.rs` 不直接维护 `TRANSCRIPT_SCROLL_STEP`、`TRANSCRIPT_MOUSE_SCROLL_STEP`、`RESULT_SCROLL_STEP`、`RESULT_MOUSE_SCROLL_STEP`、`handle_transcript_scroll_key`、`handle_result_scroll_key`、`scroll_result_from_mouse`、`scroll_result`、`scroll_result_down`、`result_scroll_event`、`result_output_line_count`、`scroll_transcript` 或 `transcript_scroll_event`。
- monitor quick action 的键盘选择、事件文案、edit-before-run、提交和点击激活由 `src/ui/quick_actions.rs` 拥有；`src/ui.rs` 不直接维护 `handle_monitor_quick_action_key`、`selected_quick_action_event`、`activate_selected_monitor_quick_action` 或 `activate_monitor_quick_action_at_row`。
- 粘贴事件路由、粘贴换行归一化、active prompt/filter/message box 写入和 paste event 文案由 `src/ui/paste.rs` 拥有；`src/ui.rs` 不直接维护 `handle_tui_paste` 或 `normalize_pasted_text`。
- 主 TUI terminal mode setup/teardown 命令列表由 `src/ui/terminal_mode.rs` 拥有；`src/ui.rs` 不直接维护 terminal mode helper 或滚轮捕获策略。
- clipboard 快捷键路由、输入框鼠标/键盘选区复制、OSC 52 请求和复制事件文案由 `src/ui/clipboard.rs` 拥有；`src/ui.rs` 不直接维护 `handle_clipboard_key` 或 `handle_input_selection_mouse`。
- 矩形命中和面板内容行命中 helper 由 `src/ui/geometry.rs` 拥有；`src/ui.rs` 不直接维护 `rect_contains` 或 `rect_content_row_contains`。
- UI 大型单测由 `src/ui/tests.rs` 拥有；`src/ui.rs` 不直接保留 inline `mod tests { ... }`。
- task monitor shell 的文本拼装、tab strip、快捷操作聚合、内部命中 helper 和面板截断由 `src/ui/monitor_shell.rs` 拥有；`src/ui.rs` 不直接维护 `render_task_monitor`、`format_task_monitor_text`、`format_task_overview_lines`、`monitor_quick_actions_for_tab`、`monitor_tab_strip`、`format_monitor_tabs`、`select_monitor_tab_at_position`、`clicked_monitor_quick_action_index`、`visible_panel_line_indices`、`truncate_panel_lines`、`truncate_panel_lines_with_focus` 或 `selected_monitor_quick_action_line`。
- 运行中本地命令的解析、read-only/write-output guard、状态/BTW/Terminal/Git 旁路处理和 unsupported/deferred 提示由 `src/ui/running_commands.rs` 拥有；`src/ui.rs` 不再直接维护这组命令分发和 safety gate。
- resume picker 的过滤、独立选择循环、TUI 内键鼠处理、列表/预览布局和预览文本由 `src/ui/resume_picker.rs` 拥有；`src/ui.rs` 不直接拼接 resume picker 的列表或预览内容。
- 输入区 permission dialog 的 blocker 投影由 `src/ui/dialogs.rs` 拥有；legacy Approvals formatter 的审批/旁路问题选择、鼠标命中、批准/拒绝和 session/runtime 写回由 `src/ui/approvals.rs` 拥有；`src/ui.rs` 不直接维护 blocker 选择或审批写回逻辑。
- UI 投影应来自运行时、会话与命令报告模型，而不是解析终端文本。
- 高风险操作应预填或经由审批路由，而不是由含糊的 UI 点击触发。
- monitor 标签分为 core 与 advanced 两档（`src/ui/monitor.rs` 中的 `MonitorTabMetadata`）：core 任务视图（Overview/Changes/Tools/Tests/Session/Approvals/Context）始终占据主标签条，其中 Usage/Deliver/Tests/Session/Context/Environment/Approvals formatter 由 `src/ui/monitor.rs` 拥有，直接消费 `SessionMonitor` projection 展示用量、交付、测试、会话、上下文、环境、审批和旁路问题状态；advanced/support 诊断视图（Result/Usage/Health/Library/Deliver/Environment/Trace）默认折叠，仅以一个 `+advanced` 开关入口出现；当某个 advanced 标签成为活动标签时才在分隔符后展开整组 advanced 标签。标签顺序、label 与 tier 由 `MONITOR_TAB_METADATA` 单一来源定义，`MonitorTab::all()`、`tier()`、`label()`、`next()`/`previous()`、标签条渲染（`monitor_tab_strip`）与点击命中都从该 projection 派生，避免多处各写一份顺序或名称。
- monitor 静态快捷操作由 `src/ui/monitor.rs` 中的 `MONITOR_STATIC_QUICK_ACTIONS` / `MonitorTabQuickActions` 投影定义，覆盖 Overview/Result/Changes/Usage/Tests/Session/Approvals/Context/Trace；运行中需要替换为当前 running handler 可执行的观察命令时，由同一 owner 的运行态 quick-action projection 提供；不依赖 `TuiState` 的 Tools/Deliver/Environment 动态操作也由 `src/ui/monitor.rs` 生成；需要读取 workspace 与配置的 Health 操作和 formatter 由 `src/ui/monitor_health.rs` 生成；需要读取 workspace 和本地库状态的 Library 操作和 formatter 由 `src/ui/monitor_library.rs` 生成，避免把状态读取硬塞进静态表。
- Changes tab 的 worktree 状态读取、Git status/diff 解析、session diff 聚合、patch 选择/滚动和鼠标命中由 `src/ui/monitor_changes.rs` 拥有；`src/ui.rs` 只保存当前选中项、滚动位置和最后事件等 TUI 状态，并调用 owner 暴露的 projection/interaction 函数。
- Tools tab 的工具日志行模型、详情预览/截断、选择/展开、可见行映射和 Ctrl-O/Ctrl-F 预填由 `src/ui/monitor_tools.rs` 拥有；`src/ui.rs` 只保存当前工具列表、选中工具和外层鼠标/键盘分发状态。
- 键盘循环（Ctrl-T / Ctrl-←/→ / Tab）仍按 `all()` 遍历全部标签，进入 advanced 标签时主条自动展开；内部命中 helper 仍可把 `+advanced` 映射到第一个 advanced 标签，并把 core 标签映射为收回 advanced 组。折叠由活动标签的 tier 派生，不引入额外 TUI 状态字段。
