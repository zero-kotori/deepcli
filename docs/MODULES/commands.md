# 命令模块

## 职责

`src/commands.rs` 目前负责命令分发、大多数一次性命令处理器、稳定 JSON 报告构建器、面向会话的报告、支持诊断、Git 身份报告以及本地工作流格式化。`src/commands/agent.rs` 负责 `/agent` 的 list/show/spawn 处理、子代理任务选择、agent inspect JSON、子代理任务投影以及 agent 后续操作。`src/commands/approval.rs` 负责 `/approval` 的 list/approve/deny/clear 处理、审批请求文本/JSON 格式化、操作后续动作以及审批输出写入。`src/commands/btw.rs` 负责 `/btw` 的 ask/list/answer/clear 处理、附带问题文本/JSON 格式化、回答选项解析、操作后续动作以及 by-the-way 输出写入。`src/commands/completion.rs` 负责 shell 补全命令目录、脚本格式化、安装/状态报告以及补全输出写入。`src/commands/config.rs` 负责 `/config` 的 show/sources/validate/get/set 处理、config inspect JSON、配置来源报告、项目配置更新以及被支持类命令复用的配置校验辅助函数。`src/commands/credentials.rs` 负责 `/credentials` 的 status/template/import/set/remove 处理、凭据文件读写、脱敏后的凭据状态 JSON、本地配置后续操作以及隐藏/标准输入方式的 API key 录入。`src/commands/context.rs` 负责 `/context` 工作区上下文收集与文本格式化。`src/commands/delivery.rs` 负责变更交付集群 —— `/diff`、`/review`、`/verify`、`/handoff` —— 包括 diff/verify/handoff 选项解析、会话 diff 源解析与路径过滤、diff stat/name-only/display 格式化、worktree review 启发式与 review 风险摘要、验证报告构建（测试证据、环境检查、阻塞项）以及 handoff 报告的 Markdown/PR/JSON 格式化；它通过 `super::` 复用 `src/commands.rs` 中的会话、env 以及共享格式化辅助函数。`src/commands/diagnose.rs` 负责 `/diagnose` 与 `/support` 的选项解析、诊断报告 JSON、脱敏后的支持包生成与产物、issue 模板以及 diagnose 后续操作，同时把工作区健康检查委托给 `/doctor` 处理器、把会话诊断委托给 `/session` 处理器。`src/commands/doctor.rs` 负责 `/doctor` 与 `/init` 处理：doctor 选项解析、工作区/shell/provider/test/Git 身份健康检查、shell 命令路径解析、provider 就绪报告与在线 provider 探测、doctor 修复应用以及 doctor 报告的文本/JSON 格式化；`handle_doctor` 被重新导出，以便 `/diagnose` 复用它。`src/commands/env.rs` 负责 `/env` 的 check/plan/setup/install/test 处理、环境选项解析、environment check/plan/setup/test JSON、environment 后续操作以及 slash-to-deepcli 命令映射；verify/handoff 辅助函数通过 `super::` 复用 `environment_checks_json`/`environment_status`/`validate_env_target`/`first_line`，而共享的 `environment_next_actions`/`dedup_preserve_order` 辅助函数仍保留在 `src/commands.rs` 中。`src/commands/fork.rs` 负责 `/fork` 源选择、会话载荷克隆、终端启动处理、dry-run/错误/验证报告、fork JSON schema 以及 fork 后续操作。`src/commands/git.rs` 负责 `/git` 读/写操作分发、Git 选项解析、dry-run Git 操作报告、Git inspect JSON、Git 输出写入以及 Git 后续操作。`src/commands/goal.rs` 负责 `/goal` 的 show/start/clear/status/gate 处理、默认目标契约创建与守护计划生成、目标就绪度收集、被 `/round` goal-status 摘要复用的目标会话选择，以及 goal 文本/JSON 格式化。`src/commands/help.rs` 负责命令帮助主题、帮助文本格式化、主题别名归一化以及帮助摘要。`src/commands/logs.rs` 负责 `/logs` 日志发现、安全 tail、脱敏、文本/JSON 报告以及 logs 输出写入。`src/commands/model.rs` 负责 `/model` 的 show/list/set 处理、模型检查 JSON、provider 运行时展示以及项目模型配置更新。`src/commands/opportunities.rs` 负责 `/opportunities` 选项与过滤器解析、scorecard 产品机会过滤与后续操作，以及 opportunity 文本/JSON 格式化；与 `/recipes` 一样，它通过 `super::` 复用 `src/commands.rs` 中的 `build_round_report` 与 scorecard 投影辅助函数。`src/commands/parser.rs` 负责斜杠命令解析、公开的斜杠命令变体、别名以及参数归一化。`src/commands/permissions.rs` 负责 `/permissions` 的 show/set-mode 处理、权限策略 JSON 以及项目权限模式更新。`src/commands/plan.rs` 负责 `/plan` 的 show/draft 处理、澄清问题生成、需求草稿 Markdown/JSON 格式化、附带问题队列写入以及 plan 输出写入。`src/commands/preflight.rs` 负责 `/preflight` 检查规划、执行摘要、诊断以及发布检查 JSON。`src/commands/privacy.rs` 负责 `/privacy` Git 历史/隐私扫描、敏感路径与 token 检测、屏蔽词抑制、脱敏后的隐私文本/JSON 报告以及隐私扫描输出写入。`src/commands/productloop.rs` 负责产品循环核心三件套 —— `/scorecard`、`/round` 与 `/benchmark` —— 包括 scorecard 报告构建器与机会项、round 报告构建器与 goal/benchmark 闸门、完整的 benchmark 套件（run/record/status/summary/trends/compare/baselines/show/cleanup）、SOTA 基线后续操作与 benchmark-baseline 辅助函数，以及稳定的 `deepcli.scorecard.v1`/`deepcli.round.v1`/`deepcli.benchmark.*` JSON schema。它重新导出 `build_round_report`、`RoundReport`、`ScorecardOpportunity`、`scorecard_*` 投影辅助函数、`sota_baseline_next_actions`、`local_action_checklist` 以及 `DEFAULT_ROUND_SCORE_THRESHOLD`/`DEFAULT_BENCHMARK_*` 常量，以便 `/recipes`、`/opportunities` 及其他模块能继续通过 `super::` 使用它们。`src/commands/prompt.rs` 负责 `/prompt` 的 list/get/render/save/delete 处理、prompt inspect/render JSON、prompt render 参数投影、prompt 输出写入以及 prompt 后续操作。`src/commands/quickstart.rs` 负责 `/quickstart` 引导检查、就绪 JSON 以及启动后续操作。`src/commands/recipes.rs` 负责 `/recipes` 主题归一化、recipe 目录、SOTA 产品循环 recipe 状态以及 recipe 文本/JSON 格式化；在产品循环集群被抽取出来之前，它通过 `super::` 复用 `src/commands.rs` 中的 `build_round_report`、SOTA 基线后续操作以及 scorecard 机会投影辅助函数。`src/commands/registry.rs` 负责命令分组元数据、帮助摘要类型以及 running-safe 分类。`src/commands/resume.rs` 负责 `/resume` 的 preview/candidates 处理、可恢复会话过滤、resume JSON schema、输出写入以及 resume 后续操作。`src/commands/response.rs` 负责共享的命令输出辅助函数、限定在工作区内的 `--output` 写入以及结构化命令退出。`src/commands/selftest.rs` 负责 `/selftest` 就绪检查、本地安装诊断以及 selftest JSON。`src/commands/session.rs` 负责 `/session` 子命令分发（list/history/next/diagnose/search/rename/export/prune-empty/tools/trace/restore-backup 以及审批/附带问题队列）、running-safe 的 `/session` 处理器、restore-backup 预览/应用、可恢复会话选择与降噪、会话活动/检查/诊断 JSON 以及会话后续操作投影。它重新导出会话选择/投影辅助函数（`session_metadata_json`、`resolve_session_for_*`、`sessions_with_resumable_context`、`short_id`、`SessionFallbackKind` 等），以便 `/resume`、`/fork`、`/approval`、`/btw` 以及 verify/handoff 辅助函数能继续通过 `super::` 使用它们。`src/commands/skill.rs` 负责 `/skill` 的 list/generate/run 处理、项目 skill 加载、skill inspect/run JSON、skill 输出写入以及 skill 后续操作。`src/commands/status.rs` 负责 `/status` 工作区/会话状态报告、status JSON 格式化、会话后续操作投影以及 status 输出写入。`src/commands/terminal.rs` 负责 `/terminal` 的 dry-run/open 处理、终端应用推断与校验、终端命令格式化、工作区打开后续操作以及被 fork 辅助函数共享的 terminal JSON/文本报告。`src/commands/test.rs` 负责 `/test` 的 discover/run 处理、test 命令发现/运行 JSON、被环境报告复用的 test 命令投影、test read/run 参数解析以及 test 后续操作。`src/commands/timeout.rs` 负责 `/timeout` 的 show/set/reset 处理、timeout JSON 以及 provider 轮次超时配置更新。`src/commands/trace.rs` 负责 `/trace` 会话审计选择、回退行为、审计 trace 格式化、脱敏后的 trace JSON 以及 trace 输出写入。`src/commands/usage.rs` 负责 `/usage` 会话用量报告、审计用量摘要、诊断、文本/JSON 格式化以及 usage 输出写入。`src/commands/version.rs` 负责 `/version` 文本与 JSON 报告。`src/commands/web.rs` 负责 `/web` 搜索参数归一化与 web 搜索工具分发。

## 边界

- 命令可以归一化别名并校验参数，但持久化行为应当向相关领域模块迁移。
- 命令处理器不应直接修改会话文件，只能通过 `SessionStore` 进行。
- 命令处理器不应在工具层与权限层之外执行 shell、Git、文件系统写入或网络操作。
- 新增的公开命令必须更新 `docs/COMMANDS.md` 以及针对性的命令契约测试。
- 通过 `CommandHelpSummary` 暴露的命令分组元数据必须与 `docs/COMMANDS.md` 保持同步。
- 旧版别名应保持为规范命令之上的薄封装。

## 文档同步

当命令名称、分组、归属、别名、running-safe 影响、稳定 schema 归属或兼容性发生变化时，更新 `docs/COMMANDS.md`。如果命令归属从 `src/commands.rs` 中迁移出去，则更新本文件。
